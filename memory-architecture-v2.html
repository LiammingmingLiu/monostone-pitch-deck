Phase 2
索引层 — Dual Index
两套索引并行：结构化精确匹配 + Summary 语义搜索。不对原始口语文本做 Embedding。
🎯 索引 A — 结构化（精确匹配）
participants "蔡哥" → [seg_001, seg_042]
entities "线性资本" → [seg_001, seg_003]
date 2026-03-22 → [seg_001..seg_005]
topic "融资" → [seg_001, seg_003, seg_078]
project "MVP" → [seg_002, seg_015, seg_089]
技术：SQLite / PostgreSQL / 倒排索引
查询：O(1) 精确命中，毫秒级
🧠 索引 B — 语义（模糊匹配）
Embedding 对象 Summary（非原文）
向量维度 1536d (text-embedding-3)
每个 Segment ~200字 Summary → 1 vector
适用查询 "之前聊过的融资相关的事"
不适用 "蔡哥说了什么"（用索引A）
技术：pgvector / Qdrant / Pinecone
关键：对 Summary Embed，不对原文 Embed
↓

核心原则：模型贵、存储便宜、检索极快。所以 → 写入时用模型做一次性预处理（切分+提取+Summary），检索时用数据库（零模型调用），只在最后生成时调用模型。

500小时录音 ≈ 3000-5000 Segments → 双轨检索 <100ms → 组装 3-5K tokens → 模型生成 → 3秒出结果
