Day 1
冷启动
证明即使第一天用，AI 就已经比普通 ChatGPT 有用
(a) 冷启动过程

用户口述或上传基础信息（"我叫明明，在做一个 AI 硬件创业项目，目前在融 Pre-seed"），或上传几份基础资料（PRD、团队介绍）。展示这些信息怎么进入 Memory。

(b) 第一段录音 + 提问

上传一段录音（比如一次投资人会议）。录音处理完后，问一个问题，看 AI 的回答。

(c) 和没有 Memory 的对比

同样的问题分别在 ChatGPT 和我们的系统里问。ChatGPT 需要你手动喂 Context，我们的直接能回答。

达标标准

✅ 应该能做到
记住用户名字、职业、项目名
能正确转写、切分、提取
能基于刚上传的录音回答问题
输出质量 ≈ GPT + 录音 Context，比裸 GPT 好
⬜ 不需要做到
不需要知道偏好和风格
不需要跨录音整合
不需要特别精准或个性化
不需要有个人风格
"嗯，比录音笔好用。"
Day 7
AI 开始认识你
证明 Memory 在成长，AI 开始有个性化能力
(a) 多段录音积累后的效果

系统里已有 7-10 段不同录音（投资人、团队会议、供应商等）。问一个需要跨录音整合的问题（如"帮我写产品定义报告"），展示输出融合了多段不同录音的信息。

(b) 人物识别与关联

系统能认出反复出现的人（"蔡哥"、"Harry"、"林啸"）。新录音进来后提示"你今天又和 Harry 聊了，上次你们聊的是 XXX"。人物之间有关联（Harry = 线性资本 = 投资人）。

(c) Prompt 调整 / 偏好学习

用户对某次输出做了修改（"不要这么正式"或"报告用 bullet point"）。这个偏好被记住，下次生成自动应用。

(d) Memory Snapshot

展示用户画像页面：AI 目前知道关于你的什么。类似"明明，创业者，在做 Monostone，关注融资和产品，偏好简洁沟通..."。用户可查看、编辑、删除。

(e) 自我进化 — Day 1 vs Day 7 对比

同一个问题，展示 Day 1 和 Day 7 的输出对比。Day 7 明显更精准、更符合用户习惯。

达标标准

  • 跨录音整合：从多段录音中提取相关信息，合成一个完整回答
  • 人物识别：知道"蔡哥"是谁、和你什么关系、上次聊了什么
  • 偏好记忆：记住用户对格式/语气/内容的调整偏好
  • Snapshot：有一个可查看的用户画像
  • 成长可感知：同一问题 Day 7 输出明显优于 Day 1
"它居然记得比我清楚。"
Day 30
AI 真正懂你
证明这是不可替代的个人资产，迁移成本极高
(a) 大量数据积累

系统里有 30+ 段录音 + 多份上传文件（PRD、邮件、报告等）。Memory 里有丰富的实体网络（几十个人物、多个项目、大量事实）。

(b) 复杂任务一句话完成

输入："帮我准备和红杉的会议，他们可能会问什么"。AI 基于你之前和其他投资人聊的经验，总结出投资人常问的问题、你之前哪些回答效果好、建议的 talking points。这个输出不可能从零生成，必须依赖 30 天的 Context 积累。

(c) 主动提醒 / 洞察

AI 主动发现模式，不等用户问：

  • "你最近三周和 5 个投资人聊了，他们共同关注的问题是 XXX"
  • "上次你答应蔡哥本周发 follow-up，还没做"
  • "团队会议里林啸连续两周提到 Pipeline 延期风险"
(d) 深度个性化输出

生成的邮件、报告完全是你的风格。不需要任何提示，默认就是你喜欢的格式、语气、详细程度。和 Day 1 的通用输出形成鲜明对比。

(e) Memory 丰富度可视化

展示实体网络图（人物/公司/项目的关系）、时间线（30 天里发生了什么）、事实库（多少条已验证事实）、偏好库（积累了哪些个人偏好）。

达标标准

  • 复杂推理：基于历史 Context 做跨时间、跨人物的推理
  • 主动洞察：不等用户问，主动发现模式和提醒
  • 风格一致：输出默认就是用户的风格，不需要提示
  • 可追溯：每条信息可以追溯到原始录音来源
  • Memory 丰富度：实体网络、时间线、事实库可视化
"它比我实习生更了解我。换掉它？不可能。"
