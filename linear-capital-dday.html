04 产品 · 现有方案

市面上的解决方案都缺了关键一层

录音硬件（Plaud / 讯飞）

✅ 硬件采集能力强
❌ 转写后数据就死了——不理解、不记忆、不执行
❌ 用户自己整理笔记，效率并没有真正提升

AI 聊天（ChatGPT / Claude）

✅ 推理能力强
❌ 不认识你——每次对话从零开始
❌ 没有个人 Context，输出千人一面

AI 可穿戴（Tab / Friend）

✅ 有穿戴形态
❌ Memory 能力弱或无
❌ 没有 Voice Commander 概念，被动记录为主

Memory 服务（MemoryLake）

✅ 专业记忆系统，召回率高
❌ 数据源是最大瓶颈——需用户手动上传
❌ 没有采集硬件，Context 积累速度慢

共同的缺陷：采集、理解、执行三件事被分散在不同产品里。用户要在 3-5 个工具间跳转，整合成本极高。

第一性原理：Context 从哪来？

个人 Context 是 AI 时代最重要的基础设施。核心问题只有一个：Context 从哪来？

✍️
主动输入型

用户主动告诉 AI 自己是谁
（AI 社交 / 对话输入）

只能拿到用户愿意给的
👁️
被动标签型

平台通过行为反推偏好
（抖音推荐 / 浏览记录）

只能拿到平台能观察到的
🎙️
物理世界采集型

从真实生活中直接捕获
（录音 / 视觉 / 健康数据）

高密度 Context 只能从这里来

人每天最有价值的信息——开会说了什么、和谁聊了什么、做了什么决策——这些信息产生了就消失了，从来不会主动进入任何软件系统。
真正高密度的个人 Context 必须靠硬件从物理世界采集。软件只能拿到冰山一角。

为什么现在切语音？
🎙️ 语音

✅ 模型成熟（文字模型最强）
✅ Day 1 有即时价值
✅ 硬件可无感佩戴

📷 视觉

❌ 模型不成熟
❌ 吞吐大 / 成本高
❌ 眼镜不够无感

💓 健康

✅ 传感器成熟
❌ Day 1 无即时反馈
❌ 数据需长期积累才有意义

语音不是终局，是最优的第一步。终局 = 语音 → 健康 → 视觉依次叠加。谁先跑通第一步，谁就有权叠加后续模态。

04 产品 · 靠什么赢（下）

硬件定义权 + 不可逆的时间复利

硬件定义必须由 Context 层倒推

硬件公司（Plaud/Stream）的定义是"做录音设备"→ 单麦、4h 续航、转写完就结束。

软件公司（MemoryLake）的定义是"做 Memory"→ 没有硬件，数据源靠手动上传。

两类公司都有断层。只有从 Context 倒推硬件定义，飞轮才能闭合。

我们的硬件：四变量精密平衡

要做到全天候 Context 采集，必须同时平衡：收音质量 × 音频压缩 × 通信稳定 × 高续航

四个变量互相制约——高采样率吃功耗，压缩省带宽但增加计算负载，通信要稳定要更多发射功率。在 30mAh 电池的戒指里同时做到 12-15h + 双麦高质量收音，是精密系统工程。

个人 Context = 深度壁垒，不是规模壁垒

Day 1 — AI 知道你叫明明，在做硬件创业。5 分钟就能给。

Day 7 — AI 知道你和 5 个投资人分别聊了什么、写邮件喜欢什么语气。

Day 30 — AI 知道你每个决策节点的思考路径、团队谁擅长什么、哪个投资人对哪个问题敏感。这 30 天的生活经历本身不可复制。

传统数据壁垒（规模）：我有 1000 万条数据 → 你砸钱也能到 → 可破

个人 Context 壁垒（深度）：30 天的生活轨迹、人际关系、决策偏好 → 迁移格式不兼容、语义丢失 → 不可破

迁移成本随时间指数增长。先发 6 个月 = 追不上。

🎙️ 硬件采集 → 🧠 Context 积累 → 💡 输出精准 → ❤️ 用户依赖 → 📈 更多数据 → ⚙️ 迭代 ↻

赢不是赢在某一个环节，是赢在三件事构成的飞轮转起来以后别人追不上：
押对模态和路径 × 同时控制采集和理解 × 时间复利指数增长

04 技术 · 方法论

技术方法论：写入时智能，检索时极快

核心原则：模型贵、存储便宜、检索极快。所以——写入时用模型做一次性预处理，检索时零模型调用。

写入时（一次性，用模型）
① 转写后 LLM 做主题切分（45min 录音 → 3-5 个独立 Segment）
② 每个 Segment 提取实体、事实、时间信息
③ 生成 200 字高质量 Summary
④ 对 Summary（不是口语原文）做 Embedding
检索时（零模型调用，极快）
① 结构化索引精确匹配（人名/日期/项目 → 毫秒级）
② Summary Embedding 语义搜索（模糊查询）
③ 双路合并去重 → Top-K 最相关 Segments
④ 高相关给全文，次相关给 Summary，不相关跳过

为什么不对原文做 Embedding：口语化文本语义密度低（"嗯对对对那个就是说"），Embedding 向量被稀释。对模型生成的 Summary 做 Embedding，效果翻倍。

06 风险

最大的风险及应对措施

🔴 硬件量产风险

戒指结构复杂，良率不确定，供应链交付延迟

应对：

Marshal（高通/OPPO）+ Jim（Sony）+ Ken（蔚来）= 合计 40+ 年消费电子量产经验。FPC 板 4 月中已出，量产验证在进行中。

🔴 大厂竞争风险

苹果/三星/大厂可能做类似产品

应对：

① 大厂做国内，我们出海避开直接竞争。② Memory 深度壁垒：30 天积累 = 不可迁移。③ 大厂做通用，我们做垂直深度。④ 速度：创业公司迭代速度是大厂 10x。

🟡 PMF 不确定性

用户是否愿意为 AI 记忆戒指付费

应对：

独立站 + Content 快速验证，不押重注在众筹。Plan B：iOS App 内置手机麦克风，没有戒指也能完整体验。硬件是最优解，不是唯一入口。

🟡 AI 成本风险

模型推理成本可能侵蚀利润

应对：

① Memory V2 架构降低 Token 60-80%。② BYOK 模式：Max 用户自带 Key，AI 成本用户承担，毛利率 >85%。③ 模型成本每年下降 50%+ 是确定性趋势。
