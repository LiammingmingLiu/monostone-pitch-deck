Day-1 痛点 · 介入频率

想用 AI 的场景里，
大多数都没有合适的 Context 收集设备。

回到第 2 页公式：介入频率 = AI 能从多少场景里听到你的 Context。
但人最高价值的 context 大多发生在线下、在嘴里、在对话里 —— 今天没有任何设备能无感、高频、全场景地把它捕捉下来。

🚶 散步 / 通勤

一个 idea / 一段思考冒出来 → 掏手机太慢、开声音不礼貌 → 那段话没被记下来，idea 死在路上。

🤝 多人会议

桌面录音设备 → 距离一变就转写不准；耳机录音 → 戴上别人不愿对你说话；多人发言交叉 → 谁说的什么搞不清。

💬 跟同事聊出 idea

非正式聊天 → 不能掏录音设备（社交压力）→ 那段对话最有信息含量，但全部漏掉。

💡 独自灵感时（洗澡 / 开车 / 睡前）

一段完整的思路在脑子里跑了 5 分钟 → 等坐回桌前已经忘了一半 → 最高质量的独立思考从未被沉淀过。

每一段没被捕捉的对话和思考，都是一次 context 漏掉。
没有"无感、高频、全场景"的 capture 设备 → AI 永远只能从你打字告诉它的那一小部分中理解你。

硬件 · 我们做了什么

让戒指在任何场景下都能稳定收音 + 长时间在线。

① 双麦克风 + IMU 抖动算法

戴在手上的设备最大的难题是手在动 —— 距离嘴变、角度变、风噪变。
放了 2 个麦克风 + 1 个手势识别传感器，AI 实时融合"声音处理 + 动作数据"，跟踪发声方向，动态去噪。

实验室 5 米 · 正常对话 2-3 米 · 多人场景准确率 >95%
② 双芯片架构（核心创新）

戒指电池极小，普通单芯片戒指连续录音撑不到 4h。
我们做了双芯片解耦：DSP 极低功耗专做录音 + 信号处理；BLE 极低功耗专做传输。

20mAh 电池 → 连续在线录音 12-15h · 离线缓存 4h+
③ 主动触发 · 不是 always-on

美国 30+ 州法律明确要求录音对方知情。always-on 形态没法上市。
触摸 · 双击 · 滑动 · 手势 多种交互 + 线性马达震动反馈 + LED 指示灯 → 用户每次知情，合规 + 不打扰。

④ 集成度 · 没有方案商能做

戒指要"像戒指" —— 不能太厚太怪。要在这个体积里塞 2 麦克风 + 双芯片 + IMU + 触摸 + 马达 + LED + 电池。
ID + 结构 + 堆叠 + 嵌入式 全部自研 —— 因为没有方案商做过。

找了 13 家板厂，11 家直接拒绝，只有 2 家能做。

软件 · 我们做了什么

让录音 → 可执行的 Context。
不是简单的"语音转文字"。

① 文字识别 ≠ 语义识别

目前主流 ASR 方案的问题：要么慢，要么没语义。直接送大模型则又慢又贵。
我们在 ASR 之上做了针对性处理 —— 转文字时同时保留语义，意图理解准确率显著高于通用方案。

② 多人对话 · 声纹分离 + 时间片重组

真实对话场景：你说话、我说话、并行送模型 —— 谁说的什么、顺序如何，很容易乱套。
做了声纹识别 + 数据流并发处理 + 时间片重组 → 保证每个人的话连贯、归属准确。

③ 个人 Memory 系统（核心差异化）★

市面上的录音转写产品，做到 summary + todo 就停了。
我们在中间做的事：把大量个人信息沉淀成结构化记忆。

  • ✓ 自研 Memory 模型（Tree + Graph 双结构）
  • ✓ 不是直接调大模型，针对个人场景训练
  • ✓ 越用越懂你 —— 意图理解 + 偏好学习

效果：

  • 📈 意图理解准确率持续提升
  • 📉 Token 消耗大幅下降（不重复喂 context）
  • 🔌 通过 MCP 喂给任意 agent / 大模型

这是后面所有 AI 时代愿景的根基。
